## 1. 产品概述

旅行策展人 Demo 是一个桌面优先的本地 WebApp，用广州黄金评论数据集在 `localhost` 跑通“评论 → event → route.md → 底片图 → 交互地图”的完整闭环。\
本阶段目标不是接飞书生产链路，而是验证数据预处理、AI 引擎编排、确定性 P5 交互层，以及未来零自建后端接入的边界是否清晰。

* 解决问题：把零散图文评论稳定地转成可审阅、可交互、可追溯的旅行地图作品。

* 目标用户：Demo 操作者、内部评审、后续会接手飞书接入的开发者。

* 业务价值：先用最小闭环验证“数据质量闸门 + AI 生成链路 + 前端作品展示”的可行性，再平滑切到飞书存储与留痕体系。

## 2. 核心功能

### 2.1 用户角色

| 角色     | 进入方式      | 核心权限                              |
| ------ | --------- | --------------------------------- |
| 作者/操作者 | 本地打开 Demo | 配置地图、选择评论、触发预处理与生成、在二次确认页重生成或确认保存 |
| 访客/评审  | 浏览器访问本地页面 | 浏览个人主页、进入动态地图页、沿旅程轴查看评论卡与原图       |

### 2.2 功能模块

1. **个人主页**：顶部导航、作者信息、地图卡片网格、进入地图详情。
2. **作者工作台**：地图名称/城市/风格配置、广州评论选择区、生成入口、样本状态提示。
3. **二次确认页**：底片图预览、修改 Prompt、是否基于旧图修改、重新生成/确认保存。
4. **动态地图页**：地图底片展示、底部旅程轴、右侧常驻评论卡、原图大图预览。
5. **测试追踪页**：Run 列表与 6 类留痕的结构化占位，仅做壳层与依赖留痕展示。

## 3. 核心流程

作者在工作台选择广州评论样本后，系统先运行 Part1 规则脚本，生成结构化的第②套 event 数据。随后文本模型补全广州知识并生成 `route.md`，生图模型输出年轻卡通风底片图。作者可以在二次确认页通过自然语言微调并重生成，确认后进入动态地图页。动态地图页不在底图上做热区识别，而是通过底部旅程轴与右侧常驻评论卡的 `event_id` 确定性绑定来保证可追溯。

```mermaid
flowchart LR
    A["作者在工作台选择广州评论样本"] --> B["Part1 规则脚本输出 event JSON"]
    B --> C["P1 使用 DOUBAO 做广州知识补全"]
    C --> D["P2 使用 DOUBAO 生成 route.md"]
    D --> E["P3 使用 SEEDREAM 生成年轻卡通底片图"]
    E --> F["作者在二次确认页审阅结果"]
    F --> G["触发 P4 重生成"]
    F --> H["确认保存地图"]
    G --> H
    H --> I["P5 确定性模板渲染动态地图页"]
    I --> J["访客沿旅程轴查看评论卡与原图"]
```

## 4. 用户界面设计

### 4.1 设计风格

* 主色：`#173F7A`（城市深蓝）

* 辅色：`#74D7F7`（导览天蓝）

* 强调色：`#FF7A45`（路标橙）

* 背景色：`#F7F1E3`（纸本米白）

* 卡片色：`#FFFDF8`

* 文本主色：`#16202A`

* 按钮风格：大圆角矩形、轻微抬升阴影、悬停时有“被点亮”的导览感

* 字体策略：标题使用有个性的中文展示黑体；正文使用现代中文无衬线；时间、事件编号使用等宽数字字体

* 布局风格：桌面优先，宽屏双栏与横向旅程轴并存；信息架构强调“地图 + 轨迹 + 档案卡”三分法

* 图标建议：圆润地图钉、编号印章、卡通化事件主体图标、轻量手绘导览线

### 4.2 页面设计概览

| 页面名称  | 模块名称   | UI 元素                                |
| ----- | ------ | ------------------------------------ |
| 个人主页  | 顶部导航   | 固定导航、清晰页签、当前页高亮、桌面宽屏横排               |
| 个人主页  | 作者信息区  | 大头像、昵称、旅行统计、轻档案馆标签体系                 |
| 个人主页  | 行程卡片网格 | 4 列卡片网格、底片缩略图、行程名、天数与站点数             |

| 测试追踪页 | 主从结构   | 左列表右详情；右侧采用 6 段式留痕容器，未接真实数据时也能表达未来结构 |

### 4.3 响应式策略

* 采用 **desktop-first** 设计，主设计宽度以 `1440px` 左右为基线。

* `1200px` 以上保持完整双栏/三栏布局。

* `900px ~ 1199px` 时允许部分模块纵向折叠，但不改变主流程顺序。

* 移动端仅做基础可浏览适配，不承诺完整工作台与追踪页操作体验。

* 所有关键操作保持清晰焦点态，支持键盘导航与减少动画模式。

### 4.4 本期视觉记忆点

* 动态地图页的**底部旅程轴**是整站最强识别元素，必须像一条真实导览带，把 AI 底片与原始评论牢固绑在一起。

* 右侧**常驻评论卡**要体现“可追溯档案”的感觉，而不是聊天弹窗。

* 工作台和地图页的视觉语言应共享同一套颜色与编号体系，让“生成前”和“生成后”看起来属于同一个产品。

# 前端改造设计：概念图对齐专项

## 1. 目标

- 任务分支：`feat-frontend-refactor`
- 视觉依据：飞书文档 `https://bytedance.larkoffice.com/docx/FTqndpmyko8LnrxaojTceFBRnlP`
- 改造目标：让当前 Demo 的 5 个前端页面尽量贴近文档中的 PC 概念图与 3 张风格例图，同时保持现有业务逻辑、路由、状态管理、API 调用和生成链路不变。

## 2. 已确认决策

- 采用“共享壳层重构”方案，而不是单纯 CSS 覆盖或 5 页完全独立重写。
- 第一轮允许补齐静态示意图，以提升与概念图的一致性。
- 仅改前端展示层；不修改 `store` 数据结构，不改后端接口，不改文件仓库和 AI 生成链路。
- 概念图中出现但当前系统没有真实能力支撑的控件，只做展示态或只读态，不伪造新业务能力。

## 3. 页面范围

本次纳入改造的 5 页：

1. `个人主页`
2. `动态地图页`
3. `作者工作台`
4. `静态图二次确认页`
5. `测试追踪页`

## 4. 整体实现策略

### 4.1 共享壳层

重做现有 `SiteShell`，形成统一的 PC 宽屏页面骨架：

- 页面背景改为浅暖灰白，而非当前高存在感的彩色渐变底。
- 内容区维持宽屏中心布局，顶部为产品名 + 3 个 tab 的统一导航。
- 标题区从“强运营感的 Demo 面板”改为“极简文档式页面头”，减少徽章、渐变条和强对比说明块。
- 各页的主要差异通过页面主体布局体现，不再靠统一大卡片容器包裹一切内容。

### 4.2 页面级骨架

- `个人主页`：顶部通栏导航 + 横向作者信息条 + 4 列作品网格
- `动态地图页`：左侧主画布 + 底部旅程轴 + 右侧常驻信息栏
- `作者工作台`：左栏配置 + 左栏底部风格预览 + 中右评论选择大区
- `静态图二次确认页`：左侧大图预览 + 右侧操作侧栏
- `测试追踪页`：左侧 run 列表 + 右侧详情卡区 + 底部追溯胶囊链

### 4.3 功能保持原则

- 保留现有页面路由和 URL 结构。
- 保留现有按钮点击事件、`fetch` 行为、`router.push/refresh`、`useWorkspaceStore` 使用方式。
- 保留动态地图页的 `event_id` 选中逻辑与大图预览弹层。
- 保留测试追踪页当前“壳层页”定位，不新增真实留痕逻辑。

## 5. 视觉系统

### 5.1 色板

从当前蓝橙高饱和方案，收敛为更贴近概念图的白色极简体系：

- `--bg-page`: 浅暖灰白，用于整页背景
- `--bg-surface`: 纯白，用于主要卡片、面板和弹层
- `--bg-soft`: 很浅的米灰，用于次级区块、输入底和选中弱态
- `--text-strong`: 深墨色，用于标题和主要正文
- `--text-muted`: 中灰棕，用于说明、标签和弱信息
- `--line-subtle`: 低对比细线边框
- `--accent-primary`: 低饱和珊瑚红，仅用于 CTA、选中态、重点数字或高亮标记

不再使用当前大面积蓝色渐变封面、强青色描边和重强调色按钮体系。

### 5.2 组件风格

- 边框：细、浅、稳定
- 阴影：弱阴影，只承担层级关系，不制造悬浮感
- 圆角：保留圆角，但控制在更克制的区间，不继续走厚卡片感
- 留白：增大模块间距和内部 padding，让页面更接近概念图的“宽、松、静”
- 文案层级：标题、辅助文本、标签、按钮文案重新分级，减少当前过多的全大写 eyebrow 和高强调说明块

### 5.3 字体策略

- 不引入新的远程字体，继续使用当前本地字体链，避免额外加载风险。
- 通过字号、字重、字距和行高重新建立层级：
  - 页面标题：更大但更轻一点
  - 卡片标题：清晰、稳重
  - 辅助说明：更小、更淡
  - 导航与标签：统一到更克制的样式
- 注意在前端不能暴露app内部机密信息，比如大模型型号、密钥、和不应当展示给用户的内部机制信息（举例：底图只做展示；真正的交互发生在底部旅程轴与右侧常驻评论卡之间，节点和评论通过 event\_id 一对一绑定。）

## 6. 5 页映射细节

### 6.1 个人主页

目标形态：

- 顶部是品牌名和 3 个 tab
- 作者信息横向展开，头像、名字、旅行次数、足迹数同区展示
- 作品区改为 4 列宽屏卡片网格

保持不变：

- 地图卡仍由现有 `MapRecord[]` 驱动
- 卡片点击仍进入对应地图页
- 状态标签仍由现有状态字段驱动

展示改造：

- 卡片封面优先使用静态示意封面或真实 poster 的安全回退，不改变卡片的业务绑定
- 当真实数据不足时，不伪造业务地图，只保留真实记录数量

### 6.2 动态地图页

目标形态：

- 地图画布占左侧约 70%
- 底部旅程轴横跨主画布下缘
- 右侧为常驻详情栏，承载地点图、时间、正文和风格信息

保持不变：

- 选中节点仍由 `selectedEventId` 控制
- `event_id` 到评论卡的确定性绑定不变
- 图片预览弹层逻辑不变

展示改造：

- 旅程轴从当前厚卡片列表，调整为更轻的横向轨道式组件
- 右栏增加更接近概念图的信息编排
- 若概念图中的风格选择器当前无真实切换能力，则展示为只读态（human：本页面中不支持风格选择器，删去该视觉组件）

### 6.3 作者工作台

目标形态：

- 左栏固定放配置项
- 左栏下半部分展示 3 张风格例图（human：改为选中风格的对应例图）
- 中右区域成为宽屏评论选择网格（human：单次显示4x2=8张图，更多的卡片溢出，用户向下滑动可见）
- 主 CTA 落在右下视觉焦点位

保持不变：

- `mapName` 输入、评论勾选、全选、重跑预处理、生成地图的现有逻辑不变
- 当前仅支持 `young-cartoon` 的事实不变

展示改造：

- 把 3 张风格例图作为静态展示资产接入，不用标注默认 / 待实现
- 评论卡从当前“厚卡片 + 小图”改为更接近概念图的信息卡样式

### 6.4 静态图二次确认页

目标形态：

- 大图区聚焦地图预览
- 右侧窄栏承载 prompt 修改、是否基于旧图修改、重新生成、确认保存

保持不变：

- `instruction`、`basedOnExistingImage`、重新生成和确认保存逻辑不变

展示改造：

- 右栏控件重新排版，减少当前面板感
- 预览区扩大并弱化包裹层，让地图成为绝对主视觉

### 6.5 测试追踪页

目标形态：

- 左侧约 25% 宽度显示 run 列表
- 右侧约 75% 显示当前 run 的 5 类留痕详情
- 底部增加一条追溯胶囊链

保持不变：

- 当前选中 run 的切换逻辑不变
- 当前“仅壳层”的业务边界不变

展示改造：

- 列表与详情的视觉主从更清晰
- 各类 trace 卡片统一为轻量信息面板

## 7. 静态示意图策略

新增静态素材目录：`public/ui-static/`

建议子目录：

- `public/ui-static/profile/`
- `public/ui-static/covers/`
- `public/ui-static/styles/`

使用原则：

- 风格例图直接承接文档中的 3 张原图，作为工作台左栏风格预览
- 个人主页头像、作品封面等装饰性视觉允许使用静态示意图
- 真实业务图片入口、评论原图预览、地图 poster 主图仍保持现有真实数据链路

## 9. 风险与控制

### 风险 1：概念图中的展示控件超出现有真实能力

控制：

- 只做展示态，不新增假逻辑

### 风险 2：概念图作品数量多于当前真实地图数量

控制：

- 不伪造真实作品数据
- 允许卡片封面使用静态示意图，但卡片数量仍由真实地图记录驱动

### 风险 3：共享壳层重构可能波及 5 页

控制：

- 优先改 `SiteShell` 和页面结构层
- 不改路由和行为接口
- 实施后逐页回归页面跳转、生成、确认、地图交互和 run 切换

## 10. 验证方案

实施完成后至少验证：

1. 首页、工作台、二次确认页、动态地图页、测试追踪页都能正常进入
2. 工作台生成按钮仍能触发原有流程
3. 二次确认页的重新生成与确认保存仍可用
4. 动态地图页底部节点切换仍驱动右侧详情变化
5. 评论原图预览弹层仍可打开和关闭
6. 测试追踪页切换不同 run 时详情同步刷新
7. 运行 `npm run lint`
8. 若实现过程中未触发行为逻辑调整，则测试以现有测试集通过和人工回归为主

## 11. 非目标

- 本轮不新增真实风格切换能力
- 不新增飞书运行链路
- 不重构数据契约
- 不重写生成流程
- 不为了像概念图而制造不存在的业务数据
